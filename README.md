# Sourcing Funnel Analysis with AI-Assisted Candidate Prioritization

This project analyzes a fictional sourcing/recruitment dataset to identify funnel bottlenecks, conversion patterns, candidate advancement signals, and practical recruiter recommendations.

## Files

- `sourcing_analysis.ipynb` — main analysis notebook and solution demonstration
- `sourcing_analysis_report.pdf` — short report explaining approach, decisions, AI usage, key insights, and recommendations
- `sourcing_analysis_summary.md` — written summary of the analysis
- `mock_sourcing_dataset.csv` — dataset used by the notebook
- `mock_sourcing_dataset.xlsx` — original Excel dataset
- `requirements.txt` — Python dependencies

## Questions Answered

- Which sourcing channels convert best?
- Where do candidates drop in the funnel?
- Which candidate signals indicate a higher chance of advancing?
- When should recruiters persist with a candidate?
- When is there a low probability of conversion?
- How can AI support recruiter decision-making?

## Methodology

The analysis includes:

1. Funnel conversion analysis across recruitment stages
2. Conversion comparison by sourcing channel
3. Conversion comparison by recruiter
4. Candidate signal analysis using scores, seniority, response behavior, and rejection reasons
5. AI-assisted classification of qualitative recruitment notes into structured insight categories

## AI Usage

AI was used to support the qualitative analysis of recruiter notes by grouping unstructured comments into practical categories such as salary risk, low engagement, timing issue, strong technical fit, and technical mismatch.

The AI-supported tags were used to help identify recurring candidate risks and prioritization signals. Final recommendations were based on both structured funnel metrics and human interpretation.
